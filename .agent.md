## Development Commands

### Core Development
```bash
# Start development server (port 3000, configurable via PORT env var)
pnpm run dev

# Development with specific configurations
pnpm run dev:docker          # Development on port 3002
pnpm run dev:public          # Development with public Socket.IO
pnpm run dev:full            # Development with WebSocket server
pnpm run dev:socketio        # Development with Socket.IO server

# Build and Production
pnpm run build              # Production build
pnpm run start              # Start production server
pnpm run lint               # Run ESLint

# Testing
pnpm run test:db            # Test database connection
pnpm run test:storage       # Test storage connection
pnpm run test:all           # Run all tests
```

### WebSocket/Socket.IO Servers
```bash
# WebSocket Server
pnpm run ws:start           # Start WebSocket server
pnpm run ws:dev             # Start WebSocket with nodemon

# Socket.IO Server
pnpm run socketio:start     # Start Socket.IO server
pnpm run socketio:dev       # Start Socket.IO with nodemon
pnpm run socketio:local     # Local Socket.IO development
pnpm run socketio:public    # Public Socket.IO (port 3003)
```

## Architecture Overview

### Tech Stack
- **Framework**: Next.js 14.2.15 with App Router
- **Database**: Supabase (PostgreSQL) with TypeScript interfaces
- **Storage**: Multi-tier system (Cloudflare R2, Google Drive, Local backup)
- **Real-time**: Socket.IO and WebSocket support
- **UI**: React 18 with Tailwind CSS, Radix UI components
- **Image Processing**: Sharp with automatic optimization
- **Deployment**: Docker, PM2, Railway/Render support

### Key Components

#### Database Layer (`src/lib/database.ts`)
- Centralized database service using Supabase
- Handles Events, Photos, Messages, and Pricing Packages
- Includes image optimization and compression analytics
- TypeScript interfaces for type safety

#### Storage System (`src/lib/storage-adapter.ts`)
- Smart storage management across multiple providers
- Automatic tier selection based on file characteristics
- Integration with DSLR-System for advanced storage features
- Handles uploads, optimization, and cleanup

#### WebSocket/Socket.IO Integration
- Dual real-time communication support
- Configurable via `NEXT_PUBLIC_USE_SOCKETIO` environment variable
- Separate servers for different communication protocols

#### Admin System
- Comprehensive admin dashboard with mobile responsiveness
- Event management, photo management, and user analytics
- Real-time monitoring and system health checks
- Pricing packages management with drag-and-drop ordering

### DSLR-System Integration
- Separate system for DSLR camera auto-upload
- Located in `DSLR-System/` directory
- Handles automatic photo processing and cloud storage
- Scripts for event management and storage monitoring

### Environment Configuration
- Multiple environment support (dev, staging, production)
- Environment-specific configurations in `.env.*` files
- Centralized environment management scripts
- Docker and CI/CD integration

## Development Workflow

### Local Development
1. Use `pnpm run dev` for standard development
2. For real-time features, use `pnpm run dev:socketio` or `pnpm run dev:full`
3. Environment switching via scripts in `scripts/development/`
4. Testing with `pnpm run test:all`

### Production Deployment
1. Build with `pnpm run build`
2. Use PM2 for process management: `pnpm run pm2:start`
3. Monitor with enhanced health monitoring: `pnpm run monitor:start`
4. Deploy via Docker, Railway, or Render

### Key Scripts by Category
- **Development**: `scripts/development/` - Environment setup and switching
- **Monitoring**: `scripts/monitoring/` - Health checks and system monitoring
- **CI/CD**: `scripts/ci-cd/` - Deployment and GitHub Actions
- **Utilities**: `scripts/utils/` - Various development utilities

## Important Notes

- The project uses TypeScript with strict mode enabled
- Image optimization is handled automatically via Sharp
- Multi-storage provider system requires proper environment configuration
- Admin system includes comprehensive mobile responsive design
- Real-time features can use either Socket.IO or WebSocket
- DSLR-System provides advanced photo processing capabilities
